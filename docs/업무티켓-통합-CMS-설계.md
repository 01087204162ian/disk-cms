## 0. 기획 배경 & 목표

### 🎯 문제 인식

- **업무가 사람 중심으로 흩어져 있음**
  - 회의록, 업무일지, 결재, TODO, 실적이 각각 분리
  - 중복 작업, 누락, 실수 발생 시 책임·흐름 추적 어려움
  - 경험/노하우가 사람 머릿속에만 존재

### 🎯 목표

**"업무가 흐르면 기록이 남고, 기록이 자산이 되는 CMS"**

- 업무 통일성 확보
- 중복 작업 방지
- 실수/누락 구조적 차단
- 경험/판단 기준의 축적
- 실적의 자동 산출

## 1. 핵심 개념 정의

### 🔹 업무티켓(Ticket)

**모든 업무의 최소 단위**

- 정산, 민원, 사고, 제휴, 개발, 기획 등 전부 티켓으로 통합
- 회의록 / 업무일지 / TODO / 결재 / 실적
  - 👉 모두 **"업무티켓의 부속 정보"**

### 핵심 원칙

1. **티켓 = 업무의 원자 단위**
   - 하나의 티켓 = 하나의 완결된 업무
   - 티켓이 없으면 업무가 시작되지 않음

2. **모든 활동은 티켓에 귀속**
   - 회의록 → 특정 티켓에 연결
   - 업무일지 → 티켓 상태 변경 자동 로그
   - TODO → 티켓의 체크리스트 항목
   - 결재 → 티켓 상태 변경의 일부

## 보험 운영 플랫폼 확장

### 확장 목표

"업무티켓 중심 통합형 CMS"를 **보험 운영 플랫폼**으로 확장

- **Contract(계약)** 객체 포함
- **보험사(Partner)** 포함
- **Ticket은 Contract 위에서 발생하는 모든 행위**

### 핵심 관계

```
Partner (보험사/제휴사)
  ↓
Contract (계약)
  ↓
Ticket (업무 티켓)
  ↓
Activity Logs / Approvals / Checklists
```
