## 1. 테스트 환경 준비

### 1.1 서버 접속

1. **브라우저에서 접속**
   - URL: `https://disk-cms.simg.kr`
   - 브라우저: Chrome, Firefox, Safari, Edge (최신 버전)

2. **로그인**
   - 테스트 계정으로 로그인
   - 세션이 정상적으로 유지되는지 확인

### 1.2 테스트 계정 준비

**필요한 계정**:
- 일반 사용자 계정 (티켓 생성, 수정 가능)
- 승인자 계정 (DEPT_MANAGER 역할) - 승인 테스트용
- 관리자 계정 (선택사항)

## 2. 기본 사용 시나리오

### 시나리오 1: 첫 방문자 - 가이드 페이지 확인

**목표**: 신규 사용자가 시스템을 처음 사용할 때 가이드 페이지를 통해 이해

**테스트 단계**:

1. **사이드바에서 가이드 접근**
   - [x] 좌측 사이드바 → "보험 운영 플랫폼" 클릭
   - [x] 하위 메뉴에서 "티켓 시스템 가이드" 선택
   - [ ] 가이드 페이지가 정상적으로 로드되는지 확인

2. **가이드 페이지 탐색 (데스크톱)**
   - [ ] 좌측 사이드바 목차가 표시되는지 확인
   - [ ] 3개 섹션이 모두 보이는지 확인
   - [ ] 섹션 1: "SIMG의 일하는 방법" 클릭
     - [ ] 해당 섹션으로 스크롤 이동
     - [ ] 활성 항목이 하이라이트되는지 확인
   - [ ] 하위 항목 "1. 이 문서를 만드는 이유" 클릭
     - [ ] 해당 항목으로 정확히 이동
     - [ ] 하위 항목이 활성화되는지 확인

3. **가이드 페이지 탐색 (모바일)**
   - [ ] 브라우저 창 크기를 줄여 모바일 뷰 확인
   - [ ] 사이드바가 숨겨지고 상단 탭 메뉴가 표시되는지 확인
   - [ ] 탭 메뉴로 섹션 이동이 정상 작동하는지 확인

4. **스크롤 감지 기능**
   - [ ] 페이지를 스크롤하며 이동
   - [ ] 현재 읽고 있는 섹션이 사이드바에서 자동 하이라이트되는지 확인
   - [ ] 활성 섹션의 하위 항목이 자동으로 펼쳐지는지 확인

5. **맨 위로 버튼**
   - [ ] 페이지를 아래로 스크롤
   - [ ] 우측 하단에 "맨 위로" 버튼이 나타나는지 확인
   - [ ] 버튼 클릭 시 맨 위로 부드럽게 이동하는지 확인

**예상 결과**:
- ✅ 가이드 페이지가 정상 로드됨
- ✅ 사이드바 네비게이션이 정상 작동
- ✅ 모든 섹션과 하위 항목에 접근 가능
- ✅ 스크롤 감지 및 하이라이트 정상 작동
- ✅ 반응형 디자인 정상 작동

### 시나리오 2: 티켓 목록 확인

**목표**: 티켓 목록 페이지에서 티켓들을 확인하고 필터링

**테스트 단계**:

1. **티켓 목록 페이지 접근**
   - [ ] 사이드바 → "보험 운영 플랫폼" → "케이스 목록" 클릭
   - [ ] 또는 URL: `/pages/tickets/list.html`
   - [ ] 페이지가 정상적으로 로드되는지 확인

2. **티켓 목록 확인**
   - [ ] 티켓 목록이 테이블 형태로 표시되는지 확인
   - [ ] 티켓 번호, 제목, 상태, 담당자 등 정보가 정상 표시되는지 확인
   - [ ] 빈 목록인 경우 적절한 메시지가 표시되는지 확인

3. **필터링 기능**
   - [ ] 상태 필터: "NEW" 선택 → NEW 상태 티켓만 표시되는지 확인
   - [ ] 티켓 유형 필터: "SETTLE" 선택 → SETTLE 유형만 표시되는지 확인
   - [ ] 우선순위 필터: "high" 선택 → 높은 우선순위만 표시되는지 확인
   - [ ] 여러 필터 조합 테스트

4. **검색 기능**
   - [ ] 검색창에 티켓 번호 입력 → 해당 티켓만 표시되는지 확인
   - [ ] 검색창에 제목 키워드 입력 → 일치하는 티켓만 표시되는지 확인

5. **페이징 기능**
   - [ ] 티켓이 많을 경우 페이징 버튼이 표시되는지 확인
   - [ ] 다음 페이지 클릭 → 다음 페이지로 이동하는지 확인
   - [ ] 페이지 크기 변경 (10개, 20개, 50개) → 정상 작동하는지 확인

6. **티켓 상세 페이지 이동**
   - [ ] 티켓 목록에서 티켓 행 클릭
   - [ ] 또는 "보기" 버튼 클릭
   - [ ] 티켓 상세 페이지로 이동하는지 확인

**예상 결과**:
- ✅ 티켓 목록이 정상 표시됨
- ✅ 필터링 기능이 정상 작동
- ✅ 검색 기능이 정상 작동
- ✅ 페이징 기능이 정상 작동
- ✅ 티켓 상세 페이지로 정상 이동

## 3. 티켓 생성 및 관리

### 시나리오 3: 새 티켓 생성

**목표**: 사용자가 새 티켓을 생성하는 전체 프로세스

**테스트 단계**:

1. **티켓 생성 페이지 접근**
   - [ ] 사이드바 → "보험 운영 플랫폼" → "새 케이스 생성" 클릭
   - [ ] 또는 티켓 목록 페이지에서 "새 티켓 생성" 버튼 클릭
   - [ ] 티켓 생성 폼이 표시되는지 확인

2. **필수 필드 입력**
   - [ ] 티켓 유형 선택: "SETTLE" (정산) 선택
   - [ ] 제목 입력: "2026년 1월 정산 처리" 입력
   - [ ] 설명 입력: 상세 설명 입력
   - [ ] 우선순위 선택: "high" 선택

3. **선택 필드 입력**
   - [ ] 금액 입력: 5000000 입력
   - [ ] 심각도 선택: "P1" 선택
   - [ ] 민감도 선택: "NORMAL" 선택

4. **티켓 생성**
   - [ ] "저장" 버튼 클릭
   - [ ] 성공 메시지가 표시되는지 확인
   - [ ] 티켓 상세 페이지로 자동 이동하는지 확인
   - [ ] 티켓 번호가 자동 생성되었는지 확인 (형식: SETTLE-YYYYMMDD-XXXX)

5. **생성된 티켓 확인**
   - [ ] 티켓 상세 페이지에서 입력한 정보가 정확히 표시되는지 확인
   - [ ] 상태가 "NEW"로 설정되어 있는지 확인
   - [ ] 생성자가 현재 사용자로 설정되어 있는지 확인

**예상 결과**:
- ✅ 티켓이 정상적으로 생성됨
- ✅ 티켓 번호가 자동 생성됨
- ✅ 입력한 정보가 정확히 저장됨
- ✅ 티켓 상세 페이지로 자동 이동

### 시나리오 4: 티켓 상태 변경

**목표**: 티켓의 상태를 변경하는 프로세스

**테스트 단계**:

1. **티켓 상세 페이지 접근**
   - [ ] 생성한 티켓 또는 기존 티켓의 상세 페이지로 이동

2. **상태 변경: NEW → IN_PROGRESS**
   - [ ] 상태 변경 버튼 영역 확인
   - [ ] "작업 시작" 또는 "IN_PROGRESS" 버튼 클릭
   - [ ] 상태가 "IN_PROGRESS"로 변경되는지 확인
   - [ ] 담당자(Owner)가 현재 사용자로 설정되는지 확인
   - [ ] Activity Log에 상태 변경 기록이 추가되는지 확인

3. **IN_PROGRESS 상태에서 수정**
   - [ ] "수정" 버튼 클릭
   - [ ] 티켓 정보 수정 (제목, 설명 등)
   - [ ] 수정이 정상적으로 저장되는지 확인
   - [ ] Activity Log에 수정 기록이 추가되는지 확인

4. **상태 변경: IN_PROGRESS → REVIEW**
   - [ ] "검토 요청" 또는 "REVIEW" 버튼 클릭
   - [ ] 상태가 "REVIEW"로 변경되는지 확인
   - [ ] 승인 인스턴스가 자동 생성되는지 확인 (승인 규칙에 따라)
   - [ ] Activity Log에 상태 변경 기록이 추가되는지 확인

**예상 결과**:
- ✅ 상태 변경이 정상 작동
- ✅ 상태별 권한 제어가 정상 작동
- ✅ Activity Log가 정상 기록됨
- ✅ 승인 인스턴스가 자동 생성됨

### 시나리오 5: 체크리스트 관리

**목표**: 티켓의 체크리스트를 초기화하고 체크하는 프로세스

**테스트 단계**:

1. **체크리스트 초기화**
   - [ ] 티켓 상세 페이지 → "체크리스트" 탭 클릭
   - [ ] "체크리스트 초기화" 또는 유사 버튼 클릭
   - [ ] 티켓 유형에 맞는 기본 체크리스트 항목이 생성되는지 확인
   - [ ] 체크리스트 항목들이 표시되는지 확인

2. **체크리스트 체크**
   - [ ] 체크리스트 항목의 체크박스 클릭
   - [ ] 체크가 정상적으로 표시되는지 확인
   - [ ] 체크한 사람과 시간이 표시되는지 확인
   - [ ] Activity Log에 체크 기록이 추가되는지 확인

3. **체크리스트 해제**
   - [ ] 체크된 항목의 체크박스 다시 클릭
   - [ ] 체크가 해제되는지 확인

4. **필수 체크리스트 확인**
   - [ ] 필수 항목이 표시되는지 확인 (예: 별표 또는 "필수" 표시)
   - [ ] 모든 필수 항목을 체크하지 않고 DONE 상태로 변경 시도
   - [ ] 적절한 에러 메시지가 표시되는지 확인

**예상 결과**:
- ✅ 체크리스트가 정상 초기화됨
- ✅ 체크/해제가 정상 작동
- ✅ 체크 기록이 Activity Log에 저장됨
- ✅ 필수 항목 검증이 정상 작동

## 4. 승인 프로세스

### 시나리오 6: 승인 대기 목록 확인 및 승인 처리

**목표**: 승인자가 대기 중인 승인을 확인하고 처리

**테스트 단계**:

1. **승인자 계정으로 로그인**
   - [ ] DEPT_MANAGER 역할을 가진 계정으로 로그인

2. **승인 대기 목록 접근**
   - [ ] 사이드바 → "보험 운영 플랫폼" → "승인 대기" 클릭
   - [ ] 또는 URL: `/pages/tickets/approvals.html`
   - [ ] 대기 중인 승인 목록이 표시되는지 확인

3. **승인 목록 확인**
   - [ ] 각 승인 항목의 티켓 정보가 표시되는지 확인
   - [ ] 티켓 번호, 제목, 티켓 유형 등 정보 확인
   - [ ] 승인 레벨이 표시되는지 확인

4. **승인 처리**
   - [ ] 승인할 티켓의 "승인" 버튼 클릭
   - [ ] 승인 코멘트 입력 (선택사항)
   - [ ] 승인 완료
   - [ ] 승인 상태가 "APPROVED"로 변경되는지 확인
   - [ ] 목록에서 해당 항목이 제거되거나 상태가 변경되는지 확인
   - [ ] Activity Log에 승인 기록이 추가되는지 확인

5. **거부 처리**
   - [ ] 다른 승인 항목의 "거부" 버튼 클릭
   - [ ] 거부 사유 입력
   - [ ] 거부 완료
   - [ ] 승인 상태가 "REJECTED"로 변경되는지 확인
   - [ ] Activity Log에 거부 기록이 추가되는지 확인

**예상 결과**:
- ✅ 승인 대기 목록이 정상 표시됨
- ✅ 승인/거부 처리가 정상 작동
- ✅ 승인 기록이 Activity Log에 저장됨

---

### 시나리오 7: 승인 완료 후 DONE 상태 변경

**목표**: 모든 승인이 완료된 후 티켓을 완료 상태로 변경

**테스트 단계**:

1. **전제 조건 확인**
   - [ ] 티켓 상세 페이지 접근
   - [ ] 모든 필수 체크리스트가 완료되었는지 확인
   - [ ] 모든 승인이 완료되었는지 확인 (승인 탭에서 확인)
   - [ ] 팀장 검증이 완료되었는지 확인 (필요한 경우)

2. **DONE 상태로 변경**
   - [ ] 상태 변경 버튼 영역에서 "완료" 또는 "DONE" 버튼 확인
   - [ ] 버튼 클릭
   - [ ] 상태가 "DONE"으로 변경되는지 확인
   - [ ] Activity Log에 완료 기록이 추가되는지 확인

3. **조건 미충족 시 테스트**
   - [ ] 다른 티켓에서 체크리스트 미완료 상태로 DONE 변경 시도
   - [ ] 적절한 에러 메시지가 표시되는지 확인
   - [ ] 상태가 변경되지 않는지 확인

**예상 결과**:
- ✅ 모든 조건 충족 시 DONE 상태로 변경 가능
- ✅ 조건 미충족 시 적절한 에러 메시지 표시
- ✅ 완료 기록이 Activity Log에 저장됨

### 시나리오 8: 가이드 페이지 종합 테스트

**목표**: 새로 추가한 사이드바 네비게이션을 포함한 가이드 페이지 전체 테스트

**테스트 단계**:

1. **페이지 로드 테스트**
   - [ ] 가이드 페이지 접근: `/pages/tickets/guide.html`
